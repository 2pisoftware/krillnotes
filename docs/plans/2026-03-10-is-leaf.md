# `is_leaf` Schema Constraint — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add `is_leaf: bool` to `Schema`; when `true`, any attempt to add children to a note of that schema is blocked in core and greyed-out in the UI.

**Architecture:** The constraint is a single bool on the `Schema` struct parsed from Rhai, enforced in three workspace methods (`create_note`, `move_note`, `deep_copy_note`), surfaced through the existing `SchemaInfo` Tauri DTO, and observed in `ContextMenu` and `TreeNode` drag-drop.

**Tech Stack:** Rust (krillnotes-core + krillnotes-desktop/src-tauri), React 19 / TypeScript (krillnotes-desktop/src)

**Step 3: Add `is_leaf` check to `create_note`**

In `workspace.rs`, find the block that validates `allowed_children_schemas` on the parent (around line 1316). Prepend the `is_leaf` check **inside the same `if let Some(pid) = &final_parent` block**, before the `allowed_children_schemas` check:

```rust
if let Some(pid) = &final_parent {
    let parent_note = self.get_note(pid)?;
    let parent_schema = self.script_registry.get_schema(&parent_note.schema)?;
    // is_leaf blocks all child creation
    if parent_schema.is_leaf {
        return Err(KrillnotesError::InvalidMove(format!(
            "Cannot add children to a leaf note (schema: '{}')",
            parent_note.schema
        )));
    }
    if !parent_schema.allowed_children_schemas.is_empty()
        && !parent_schema.allowed_children_schemas.contains(&note_type.to_string())
    {
        return Err(KrillnotesError::InvalidMove(format!(
            "Note type '{}' is not allowed as a child of '{}'",
            note_type, parent_note.schema
        )));
    }
}
```

```markdown
### `is_leaf: true`

When set, notes of this schema cannot have any children. All attempts to add,
move, or paste a note as a child of a leaf note are rejected by the core with
an error. In the UI, the "Add Child" and "Paste as Child" actions are greyed out
and drag-drop onto leaf notes is blocked.

Leaf notes can still be moved, copied, or deleted. Existing children (if any
were created before `is_leaf` was set) are unaffected.
```
